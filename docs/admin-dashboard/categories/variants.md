# Variants Management

## Overview

The **Variants** page is your control center for managing product variant types - Colors, Sizes, and Custom Variants. These are the building blocks for product variations.

!!! tip "Variant Building Blocks"
Variants (colors, sizes, custom types) are reusable components that can be combined when creating products.

---

## Purpose

This page helps you:

- ✅ **Create colors** for product variants (e.g., Red, Blue, Green)
- ✅ **Create sizes** for product variants (e.g., Small, Medium, Large)
- ✅ **Create custom variant types** (e.g., Material, Length, Style)
- ✅ **Manage variant categories** - Color, Size, Custom
- ✅ **Set sort orders** to control display order
- ✅ **View variant counts** by category
- ✅ **Edit or delete** individual variants

## How to Use This Page

### Creating Color Variants

!!! tip "Color Management"
Colors are essential for fashion and apparel products.

**Steps:**

1. **Click** "Colors" tab (🎨)
2. **Click** "Create Variant" button
3. **Enter color details**:
   - **Name**: Color name (e.g., "Red", "Navy Blue")
   - **Hex Code**: Color code (e.g., "#FF0000", "#000080")
   - **Sort Order**: Display position
4. **Click** "Save" to create color

!!! example "Color Creation"
Creating a new color variant for clothing products.

**Best Practices:**

| Practice                        | Why It Matters                               |
| ------------------------------- | -------------------------------------------- |
| ✅ Use **standard color names** | Easy for sellers to understand               |
| ✅ Include **multiple shades**  | Provides variety (e.g., Light Red, Dark Red) |
| ✅ Use **accurate hex codes**   | Ensures correct color display                |
| ✅ **Set sort order** logically | Colors appear in organized way               |
| ✅ **Consider lighting**        | Colors vary by lighting, document this       |

**Common Color Palettes:**

```

   ├── Light Red (#FFB3BA)

### Editing Variants

!!! info "Quick Updates"
Modify variant information as your catalog evolves.

**Steps:**

1. **Click** "Edit" button on variant row
2. **Update variant details**:
   - Change name if needed
   - Update hex code (for colors)
   - Adjust sort order
3. **Click** "Save" to apply changes

!!! warning "Display Order Impact"
Changing sort order affects how variants appear in product forms. Test changes carefully!

### Deleting Variants

!!! danger "Consider Product Impact"
Deleting variants affects products that use them. Review before deleting.

**Before Deleting:**

| Check                        | Why It Matters                         |
| ---------------------------- | -------------------------------------- |
| ✅ **Check product usage**   | How many products use this variant?    |
| ✅ **Review current stock**  | Any inventory tied to this variant?    |
| ✅ **Consider alternatives** | Is there a better variant option?      |
| ✅ **Plan migration**        | If deleting, migrate affected products |

**Deletion Impact:**

| Impact                | What Happens                              |
| --------------------- | ----------------------------------------- |
| **Variant removed**   | Variant no longer available               |
| **Products affected** | Products lose this variant option         |
| **Historical data**   | Past orders may reference removed variant |
| **Stock tracking**    | Inventory tracking for this variant stops |

**Best Practices:**

- ✅ **Don't delete variants** with active products
- ✅ **Migrate products** to replacement variants first
- ✅ **Keep old variants** inactive instead of deleting
- ✅ **Communicate changes** to team before deleting
- ✅ **Archive rather than delete** if you might need it later

**Variant Combination Flow:**

```
[Choose Variant Type]
    ↓

    ↓

┌─────────────────────────────────┐

## Best Practices

### Variant Planning

!!! tip "Think Before Creating"
Plan your variant structure before creating individual variants.

**Planning Steps:**

1. **Analyze your products** - What variations do you need?
2. **Choose appropriate variant type** - COLOR_SIZE or CUSTOM?
3. **Define required options** - What colors, sizes, or custom values?
4. **Plan display order** - How should variants appear in product forms?
5. **Consider future needs** - Will you add more variants later?

### Color Management

!!! success "Color Strategy"
Colors define product appearance and are critical for product selection.

**Color Strategy:**

| Strategy             | Description                               | Best For                                    |
| -------------------- | ----------------------------------------- | ------------------------------------------- |
| **Core Colors**      | Essential colors every product line needs | T-shirts: Red, Blue, Black, White           |
| **Accent Colors**    | Complementary colors for variety          | T-shirts: Navy, Gray, Burgundy              |
| **Seasonal Colors**  | Season-specific color options             | Summer: Bright colors, Winter: Darker tones |
| **Limited Editions** | Special colors for premium products       | Gold, Silver, Bronze colors                 |

**Color Display Order:**

```
1. Neutral Colors (White, Black, Gray) - Foundation
2. Core Colors (Red, Blue, Green) - Main options
3. Accent Colors (Navy, Burgundy, Orange) - Variety
4. Seasonal Colors - Spring/Summer/Fall/Winter - Timed availability
```

### Size Management

!!! tip "Size Consistency"
Consistent sizing across your catalog helps sellers and customers.

**Size Consistency:**

## Troubleshooting

### Variant Not Showing in Product Form

!!! warning "Availability Issue"
If a variant doesn't appear, check its status.

| Possible Cause         | Solution                                    |
| ---------------------- | ------------------------------------------- |
| **Variant not active** | Make sure variant is active/visible         |
| **Wrong category**     | Check if variant is in the correct category |
| **Sort order issue**   | Adjust sort order if variant is buried      |
| **Cache delay**        | Refresh the product creation page           |
| **Permission issue**   | Verify you have `variants:view` permission  |

### Can't Edit Variant

!!! info "Variant Locks"
Some variants may be locked due to business rules.

| Situation           | Reason                       | Solution                            |
| ------------------- | ---------------------------- | ----------------------------------- |
| **Has products**    | Variant is used in products  | Create new variant for new products |
| **Historical data** | Variant referenced in orders | Cannot delete, may not edit         |
| **System variant**  | Protected default variant    | Cannot delete system variants       |
